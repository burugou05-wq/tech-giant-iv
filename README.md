# TECH GIANT IV - Technical Documentation

本作は、1946年の創業から現代に至るまでのエレクトロニクスメーカーの興亡を描く、高度な経営シミュレーションゲームです。
このドキュメントでは、プロジェクトの構造、各システムの詳細、およびゲームエンジンの内部ロジックについて解説します。

---

## 1. プロジェクト構造 (Directory Structure)

### `/src` - ソースコードの基盤
*   **`/components`**: UIコンポーネント群。
    *   **`/tabs`**: 各メイン画面（Dashboard, Market, Research, Production等）の実装。
    *   **`/ui`**: ボタン、カード、進捗バーなどの共通UIキット。
    *   **`EventModal.jsx`**: 歴史イベントや特報（号外）を表示するドラマチックUI。
*   **`/constants`**: ゲームの静的データ。
    *   **`technology.js`**: 全技術（シャーシ、部品ユニット）のスペック、コスト、時代設定。
    *   **`events.js`**: 歴史的イベント（朝鮮戦争、バブル、リーマンショック等）の定義。
    *   **`aiCompanies.js`**: 競合他社（Philips, RCA等）の初期設定と性格。
*   **`/context`**: 状態管理。
    *   **`GameContext.jsx`**: アプリケーション全体のグローバルステートを保持。
*   **`/hooks`**: カスタムフック。
    *   **`useGameState.js`**: 15以上の状態変数（Money, RP, Inventory等）を一括管理。
    *   **`useGameEngine.js`**: タイマーを制御し、`tickProcessor` を呼び出す心臓部。
    *   **`useGameActions.js`**: プレイヤーの操作（開発、生産開始等）のインターフェース。
*   **`/logic/engine`**:
    *   **`tickProcessor.js`**: **本作の最重要ファイル**。1Tick（14日間）の全計算をオーケストレーションする。
*   **`/systems`**: 各ドメインの独立した計算モジュール。
    *   **`marketSystem.js`**: 需要、為替、販売実行、輸送費の計算。
    *   **`productionSystem.js`**: 製造ライン、故障率、品質維持の管理。
    *   **`aiSystem.js` / `aiMarketSystem.js`**: 競合他社の意思決定ロジック。
    *   **`financeSystem.js`**: 純利益、株価、固定費の算出。
    *   **`maSystem.js`**: 企業の提携・買収・合併ロジック。
*   **`/utils`**:
    *   **`gameLogic.js`**: 製品の魅力度（Appeal）計算などの純粋関数。

## 2. コア・ゲームループ (The Tick Processor)

ゲームは1Tickごとに14日間が経過します。`tickProcessor.js` 内の `processGameTick` 関数が以下の順序で処理を実行します：

1.  **歴史イベント判定**: 現在の年月に基づき `eventSystem` でイベントを発生させる。
2.  **重点方針 (Focus) 進行**: 開発中の技術ツリーやプロジェクトの進捗を更新。
3.  **市場更新**: 世界情勢に基づき需要を変動させ、為替レートを更新。
4.  **生産処理**: 全ラインで製品を製造し、在庫を更新。メンテナンス不足による故障もここで計算。
5.  **販売実行 (`executeSales`)**:
    *   市場シェアに基づき需要を分配。
    *   魅力度の高い製品から順に在庫を消化。
    *   **輸送費 (Logistics Cost)**: 海外販売（NA, EU, CN）に対し、個数ベースのコストを計上。
6.  **AI シミュレーション**: 競合他社が新製品を出し、設備投資を行い、M&Aを画策する。
7.  **財務集計**: 売上から変動費（材料・輸送）と固定費を差し引き、純利益と新株価を算出。
8.  **経験値更新**: 事業部ごとの活動量に応じてレベルアップを処理。

## 3. 主要システムの詳細仕様

### 市場システム (`marketSystem.js`)
*   **需要曲線**: 1946年を起点とし、時代背景（ベビーブーム、IT革命等）に合わせて非線形に拡大。
*   **輸送費（新要素）**: 日本国外への販売には1個あたり $3~$6 のコストがかかり、粗利を圧迫する。
*   **為替レート**: 円安・円高が海外売上の円換算額に影響を与える。

### 製品魅力度 (`gameLogic.js`)
*   **`calculateEffectiveAppeal`**:
    *   `ベース魅力度 + 部品ユニットの加算` で算出。
    *   **陳腐化 (Decay)**: 発売から時間が経つほど、市場の期待値から外れて魅力が低下する。
    *   **時代ボーナス**: その時代の最先端技術を使っているとボーナスが付与される。

### AIロジック (`aiProductSystem.js` / `aiMarketSystem.js`)
*   **Desperate Mode**: プレイヤーが首位に立つと、上位ライバルが利益を度外視した高性能・低価格攻勢を仕掛ける。
*   **Copycat Mode**: 下位メーカーがプレイヤーの人気製品の安価なクローンを市場に投入する。
*   **独占禁止法**: プレイヤーのシェアが50%を超えると、市場の自浄作用によりシェア維持コストが急増し、強制的に他社へシェアが分配される。

### 組織・財務 (`orgSystem.js` / `financeSystem.js`)
*   **事業部レベル**: レベルが高いほど生産効率とR&D効率が向上する。
*   **株価**: 直近の利益成長率と純資産額、および市場全体の景気指数から算出される。

## 4. 開発者向けTips
*   **新しい技術を追加する**: `src/constants/technology.js` にオブジェクトを追加するだけで、自動的にR&Dマトリクスに反映されます。
*   **バランス調整**: `marketSystem.js` の `baseDemand` や、`executeSales` 内の `logisticsCostPerUnit` を調整することで難易度を変更できます。
*   **デバッグ**: ブラウザのコンソールを開くと、各Tickごとの詳細な収支内訳やAIの行動ログが出力されます。
